# Custom Code

The **Custom code** section under [Tenant Settings](/docs/products/atlas-dashboard/tenant-settings)
(`/admin/settings`) lets you paste your own markup snippets into the public storefront — typically a
measurement loader or tag-manager snippet — without needing a developer to change the site.

## The two fields

| Field | Where it goes | Typical use |
|---|---|---|
| Head HTML | Near the top of the page | Your measurement or tag-manager loader |
| Body HTML | Right after the page opens | A fallback snippet for visitors with scripting disabled |

Both fields are optional. Leaving them blank means nothing extra is injected.

## Where it applies

These snippets run **only on your public storefront** — never on the admin panel or sign-in pages. This
keeps your operator tools clean and your measurement scoped to your audience.

## Good practices

- Paste only snippets you trust; this markup runs on every public page.
- Use this for measurement and tag management; for store look and feel use
  [Branding & Appearance](/docs/products/atlas-dashboard/branding-appearance), and for search-engine
  verification use [SEO Configuration](/docs/products/atlas-dashboard/seo-configuration).
- Whitespace-only entries are treated as empty, so a stray space won't inject an empty block.

If your goal is in-product analytics, you may not need custom code at all — see
[Analytics](/docs/products/atlas-dashboard/analytics) and
[App Analytics](/docs/products/atlas-dashboard/app-analytics) for the built-in dashboards.
