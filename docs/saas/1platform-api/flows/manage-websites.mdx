# Flujo: Gestionar sitios web

## Vista general

Este flujo documenta cómo gestionar sitios web a través de la 1Platform API. Cubre el listado de categorías, el registro de sitios web, la búsqueda por URL, la actualización de detalles del sitio, la generación de páginas legales y la eliminación de sitios web.

**Caso de uso:** Registra y gestiona los sitios web de tus usuarios para generación de contenido, análisis SEO, creación de páginas legales e integraciones de terceros.

**Resultado final:** Un sitio web totalmente registrado con asignación de categoría, descripción generada con IA y páginas legales completas (política de privacidad, política de cookies, aviso legal, contacto y sobre mí).

---

## Requisitos previos

| Requisito | Detalles |
|---|---|
| App API Key | Clave de aplicación (`ak-...`) provista al registrar tu app |
| User API Key | Clave de usuario (`sk-...`) obtenida al crear el usuario |
| Proveedor de IA configurado | Requerido para descripciones y páginas legales generadas con IA |

### Headers requeridos en todas las solicitudes

```
Authorization: Bearer $APP_TOKEN
x-user-token: $USER_TOKEN
Content-Type: application/json
```

## Variables usadas en los ejemplos

```bash
BASE_URL="https://api.1platform.pro/api/v1"
APP_API_KEY="ak-your-app-api-key"
USER_API_KEY="sk-user-abc123"
APP_TOKEN=""       # Obtenido en el Paso 0
USER_TOKEN=""      # Obtenido en el Paso 1
CATEGORY_ID=""     # Obtenido en el Paso 2
WEBSITE_ID=""      # Obtenido en el Paso 3
```

## Datos requeridos para crear un sitio web

### Sitio web

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `url` | string | Sí | URL del sitio web a registrar |
| `site_name` | string | Sí | Nombre visible del sitio web |
| `category_id` | string | No | ID de categoría a asignar |
| `lang` | string | No | Idioma del sitio web: `"es"` o `"en"` (por defecto: `"es"`) |
| `site_description` | string | No | Descripción del sitio web (generada con IA si está vacía) |

### Páginas legales

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `legal_email` | string | Sí | Dirección de correo de contacto (mín. 5 caracteres) |
| `legal_owner` | string | No | Nombre del titular legal (por defecto: `""`) |
| `legal_address` | string | No | Dirección fiscal/comercial (por defecto: `""`) |
| `legal_activity` | string | No | Descripción de la actividad comercial (por defecto: `""`) |
| `site_topic` | string | No | Tema del sitio web para las páginas generadas con IA (por defecto: `""`) |

## Flujo paso a paso

### Paso 0 — Obtener el token de aplicación

**Objetivo:** Autenticar la aplicación y obtener el token JWT de aplicación.

**Endpoint:** `POST /api/v1/auth/token`

**Auth:** Ninguna (este es el punto de entrada)

```bash
curl -X POST "$BASE_URL/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$APP_API_KEY'"}'
```

**Respuesta (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "App token generated successfully"
}
```

:::note
Guarda `data.access_token` como `$APP_TOKEN`. Expira en 30 minutos.
:::

**Errores comunes:**

| Código | Causa | Solución |
|---|---|---|
| 401 | API key inválida o app inactiva | Verifica que `apiKey` sea correcta y que la app esté activa |
| 422 | Campo `apiKey` faltante o mal formado | Envía `{"apiKey": "..."}` en el cuerpo |
| 429 | Límite de tasa excedido (10 req/min) | Espera y reintenta |

---

### Paso 1 — Obtener el token de usuario

**Objetivo:** Autenticar al usuario y obtener el token JWT de usuario.

**Endpoint:** `POST /api/v1/users/token`

**Auth:** `Authorization: Bearer $APP_TOKEN` (solo token de aplicación, no se requiere token de usuario)

```bash
curl -X POST "$BASE_URL/users/token" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$USER_API_KEY'"}'
```

**Respuesta (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "User token generated successfully"
}
```

:::note
Guarda `data.access_token` como `$USER_TOKEN`. Expira en 30 minutos.
:::

**Errores comunes:**

| Código | Causa | Solución |
|---|---|---|
| 401 | API key de usuario inválida o usuario inactivo | Verifica la `apiKey` del usuario |
| 422 | Campo `apiKey` faltante | Envía `{"apiKey": "..."}` en el cuerpo |
| 429 | Límite de tasa excedido | Espera y reintenta |

:::tip
Si aún no tienes un usuario, crea uno con `POST /api/v1/users` enviando `{"email": "user@example.com"}`. La respuesta incluirá la `api_key` del usuario.
:::

### Paso 3 — Agregar un sitio web

**Objetivo:** Registrar un nuevo sitio web para el usuario autenticado.

**Endpoint:** `POST /api/v1/users/websites`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/users/websites" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://mijardin.com",
    "site_name": "Mi Jardin Digital",
    "category_id": "'$CATEGORY_ID'",
    "lang": "es",
    "site_description": "Blog sobre jardineria sostenible y paisajismo urbano"
  }'
```

**Respuesta (201):**

```json
{
  "success": true,
  "data": {
    "id": "65b2c3d4e5f6a789bcdef012",
    "url": "https://mijardin.com",
    "slug": "mijardin-com",
    "active": true,
    "user_id": "507f1f77bcf86cd799439011",
    "category_id": "65a1b2c3d4e5f6789abcdef1",
    "category": {
      "id": "65a1b2c3d4e5f6789abcdef1",
      "title": "Hogar y Jardin",
      "slug": "home-garden"
    },
    "lang": "es",
    "status": "active",
    "domain": "mijardin.com",
    "site_name": "Mi Jardin Digital",
    "site_description": "Blog sobre jardineria sostenible y paisajismo urbano",
    "created_at": "2026-03-18T10:30:00",
    "updated_at": null,
    "actions": {}
  },
  "msg": "Website added successfully"
}
```

:::note
Guarda `data.id` como `$WEBSITE_ID`.
:::

**Errores comunes:**

| Código | Causa | Solución |
|---|---|---|
| 401 | Se requiere autenticación | Regenera los tokens |
| 404 | Categoría no encontrada | Verifica que el `category_id` exista |
| 409 | El sitio web ya existe para este usuario | Usa una URL diferente o busca el sitio web existente |
| 422 | Error de validación — faltan campos requeridos | Verifica que `url` y `site_name` estén presentes |
| 429 | Límite de tasa excedido (20 req/min) | Espera y reintenta |

### Paso 7 — Generar páginas legales

**Objetivo:** Generar todas las páginas legales del sitio web: política de privacidad, política de cookies, aviso legal, página de contacto y página sobre mí.

**Endpoint:** `POST /api/v1/users/websites/{website_id}/legal`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/users/websites/$WEBSITE_ID/legal" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "legal_owner": "Jardines Digitales S.A.",
    "legal_email": "legal@example.com",
    "legal_address": "Avenida Reforma 10-20, Zona 9, Guatemala",
    "legal_activity": "Blog de jardineria y paisajismo",
    "site_topic": "Sustainable gardening and urban landscaping"
  }'
```

**Respuesta (200):**

```json
{
  "success": true,
  "data": {
    "lang": "es",
    "pages": {
      "privacy_policy": {
        "title": "Politica de Privacidad",
        "content": "<h1>Politica de Privacidad</h1><p>En Jardines Digitales S.A....</p>"
      },
      "cookie_policy": {
        "title": "Politica de Cookies",

      },
      "legal_notice": {
        "title": "Aviso Legal",

      },
      "contact": {
        "title": "Contacto",
        "content": "<h1>Contacto</h1><p>AI-generated contact page...</p>"
```

El objeto `pages` contiene cada página legal con su `title` y su `content` en HTML. El `meta.slug_map` proporciona slugs amigables para URL para cada tipo de página.

:::tip
Si omites `legal_owner` o `legal_address`, la API usará IA para generar valores adecuados basados en el contexto del sitio web.
:::

**Errores comunes:**

| Código | Causa | Solución |
|---|---|---|
| 401 | Se requiere autenticación | Regenera los tokens |

| 404 | Sitio web no encontrado | Verifica el `website_id` |
| 422 | Error de validación — `legal_email` faltante o demasiado corto | Proporciona un correo válido (mín. 5 caracteres) |
| 429 | Límite de tasa excedido (20 req/min) | Espera y reintenta |
| 500 | Falló la generación con IA | Verifica la configuración del proveedor de IA |

### Paso 8 — Eliminar un sitio web

**Objetivo:** Eliminar un sitio web de la cuenta del usuario autenticado. Esta acción es irreversible.

**Endpoint:** `DELETE /api/v1/users/websites/{website_id}`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X DELETE "$BASE_URL/users/websites/$WEBSITE_ID" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN"
```

**Respuesta (200):**

```json
{
  "success": true,
  "data": null,
  "msg": "Website deleted successfully"
}
```

:::warning
Esta acción es irreversible. El sitio web y todos sus datos asociados se eliminarán de forma permanente.
:::

**Errores comunes:**

| Código | Causa | Solución |
|---|---|---|
| 401 | Se requiere autenticación | Regenera los tokens |
| 404 | Sitio web no encontrado | Verifica el `website_id` |
| 422 | Error de validación | Revisa el formato del parámetro de ruta |
| 429 | Límite de tasa excedido (20 req/min) | Espera y reintenta |

## Ejemplo completo de extremo a extremo

Flujo completo con valores ficticios consistentes.

```bash
# === VARIABLES ===
BASE_URL="https://api.1platform.pro/api/v1"

# === PASO 0: Obtener el token de aplicación ===
APP_TOKEN=$(curl -s -X POST "$BASE_URL/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "ak-prod-app-key-2026"}' \
  | jq -r '.data.access_token')

echo "APP_TOKEN: $APP_TOKEN"

# === PASO 1: Obtener el token de usuario ===
USER_TOKEN=$(curl -s -X POST "$BASE_URL/users/token" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "sk-user-maria-2026"}' \
  | jq -r '.data.access_token')

echo "USER_TOKEN: $USER_TOKEN"

# === PASO 2: Listar categorías ===
CATEGORY_ID=$(curl -s -X GET "$BASE_URL/users/categories?status_filter=active" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  | jq -r '.data[0].id')

echo "CATEGORY_ID: $CATEGORY_ID"

# === PASO 3: Agregar un sitio web ===
WEBSITE_ID=$(curl -s -X POST "$BASE_URL/users/websites" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://mijardin.com",
    "site_name": "Mi Jardin Digital",
    "category_id": "'$CATEGORY_ID'",
    "lang": "es"
  }' \
  | jq -r '.data.id')

echo "WEBSITE_ID: $WEBSITE_ID"

# === PASO 4: Buscar un sitio web por URL ===
curl -s -X GET "$BASE_URL/users/websites?url=https://mijardin.com" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  | jq .

curl -s -X PATCH "$BASE_URL/users/websites/$WEBSITE_ID" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"site_name": "Mi Jardin Digital - Edicion 2026"}' \
  | jq .

# === PASO 6: Generar páginas legales ===
curl -s -X POST "$BASE_URL/users/websites/$WEBSITE_ID/legal" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "legal_owner": "Jardines Digitales S.A.",
    "legal_email": "legal@example.com",

    "legal_activity": "Blog de jardineria",
    "site_topic": "Sustainable gardening"
  }' \
  | jq '.data.pages | keys'
```

## Idempotencia y buenas prácticas

### Idempotencia

:::warning[Sin mecanismo de idempotencia]
La API no admite un header `Idempotency-Key`. Si una solicitud de creación de sitio web falla por timeout, busca el sitio web por URL (`GET /users/websites?url=...`) antes de reintentar. Las URLs duplicadas para el mismo usuario devuelven un error **409**.
:::

### Reintentos
- Si recibes **429** (límite de tasa), espera el tiempo indicado y reintenta.
- Si recibes **500** o un **timeout**, verifica el estado del recurso antes de reintentar.
- Ante un **401** (token expirado), regenera el token y reintenta la solicitud original.

### Timeouts
- La creación de un sitio web suele completarse en **2-5 segundos** (puede tardar más si se dispara la generación de descripción con IA).
- La generación de páginas legales implica llamadas a IA y puede tardar **10-30 segundos**.
- Configura tu cliente HTTP con al menos **45 segundos** de timeout para el endpoint de páginas legales.

### Tokens
- Ambos tokens (de aplicación y de usuario) expiran en **30 minutos** (1800 segundos). Implementa la renovación automática antes de la expiración.
- Ambos tokens son requeridos en todos los endpoints de gestión de sitios web.

### Validaciones importantes
- Las URLs de los sitios web deben ser únicas por usuario.
- El `category_id` debe referenciar una categoría activa existente.
- El campo `lang` debe ser `"es"` o `"en"`.
- El campo `legal_email` es requerido y debe tener al menos 5 caracteres.
- La eliminación de un sitio web es irreversible — no hay endpoint de restauración.
