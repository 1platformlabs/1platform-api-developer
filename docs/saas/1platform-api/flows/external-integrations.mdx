# Flow: External Integrations

## Overview

This flow documents how to connect registered websites with external services through the 1Platform API: Google Search Console for SEO management and the link-building marketplace for backlink acquisition.

**Use case:** After registering a website, connect it to Google Search Console for indexing management and sitemap submission, and to the link-building marketplace for backlink opportunities.

**End result:** A website registered in Google Search Console with sitemaps configured, and listed in the link-building marketplace with ownership verified.

---

## Prerequisites

| Requirement | Details |
|---|---|
| App API Key | Application key (`ak-...`) provided when registering your app |
| User API Key | User key (`sk-...`) obtained when creating the user |
| Website registered | A website must be registered via `POST /users/websites` first |
| Google service account | A service account with Search Console access (for Search Console integration) |
| Marketplace credentials | User must have marketplace credentials configured (for link-building integration) |

### Required headers for all requests

```
Authorization: Bearer $APP_TOKEN
x-user-token: $USER_TOKEN
Content-Type: application/json
```

## Step-by-step flow

### Step 0 — Get App Token

**Goal:** Authenticate the application and obtain the app JWT token.

**Endpoint:** `POST /api/v1/auth/token`

**Auth:** None (this is the entry point)

```bash
curl -X POST "$BASE_URL/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$APP_API_KEY'"}'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "App token generated successfully"
}
```

:::note
Save `data.access_token` as `$APP_TOKEN`. Expires in 30 minutes.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Invalid API key or inactive app | Verify that `apiKey` is correct and the app is active |
| 422 | Missing or malformed `apiKey` field | Send `{"apiKey": "..."}` in the body |
| 429 | Rate limit exceeded (10 req/min) | Wait and retry |

---

### Step 1 — Get User Token

**Goal:** Authenticate the user and obtain the user JWT token.

**Endpoint:** `POST /api/v1/users/token`

**Auth:** `Authorization: Bearer $APP_TOKEN` (app token only, no user token required)

```bash
curl -X POST "$BASE_URL/users/token" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$USER_API_KEY'"}'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "User token generated successfully"
}
```

:::note
Save `data.access_token` as `$USER_TOKEN`. Expires in 30 minutes.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Invalid user API key or inactive user | Verify the user's `apiKey` |
| 422 | Missing `apiKey` field | Send `{"apiKey": "..."}` in the body |
| 429 | Rate limit exceeded | Wait and retry |

:::tip
If you don't have a user yet, create one with `POST /api/v1/users` sending `{"email": "user@example.com"}`. The response will include the user's `api_key`.
:::

### Step 2 — Add Website to Search Console

**Goal:** Register the website in Google Search Console.

**Endpoint:** `POST /api/v1/users/websites/{website_id}/searchconsole`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/users/websites/$WEBSITE_ID/searchconsole" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"action": "add"}'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "action": "add",
    "status": "completed",
    "message": "Website added to Search Console"
  },
  "msg": "Action completed successfully"
}
```

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | Invalid action or website not registered in Search Console | Verify action is `add`, `verify`, `delete`, or `sitemaps` |
| 401 | Authentication required | Regenerate tokens |
| 402 | Insufficient balance — not enough credits | Top up your account balance |
| 404 | Website not found | Verify the `website_id` |
| 422 | Validation error — missing or malformed fields | Check required fields |
| 429 | Rate limit exceeded (20 req/min) | Wait and retry |
| 502 | Google Search Console API error | Service temporarily unavailable — retry later |

### Step 4 — Submit Sitemaps

**Goal:** Submit sitemap URLs for the website in Google Search Console.

**Endpoint:** `POST /api/v1/users/websites/{website_id}/searchconsole`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/users/websites/$WEBSITE_ID/searchconsole" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "sitemaps",
    "payload": {
      "sitemaps": [
        "https://mijardin.com/sitemap.xml",
        "https://mijardin.com/sitemap-posts.xml"
      ]
    }
  }'
```

```json

  "success": true,
  "data": {

    "status": "completed",
    "message": "Sitemaps submitted successfully"
  },
  "msg": "Action completed successfully"
}
```

## Idempotency and best practices

### Idempotency
- Search Console and marketplace actions are generally idempotent — re-adding an already-added website will return an appropriate error rather than creating duplicates.

### Retries
- If you receive **429** (rate limit), wait the indicated time and retry.
- If you receive **502** (service unavailable), the external service is temporarily down — retry after a short delay.
- For **401** (expired token), regenerate the token and retry the original request.

### Timeouts
- External integration calls involve third-party APIs and may take **5-15 seconds**.
- Configure your HTTP client with at least **30 seconds** timeout for these endpoints.

### Tokens
- Both tokens (app and user) expire in **30 minutes** (1800 seconds). Implement automatic renewal before expiration.
- Both tokens are required on all integration endpoints.

### Important validations
- The website must be registered in the platform (`POST /users/websites`) before adding to external services.
- For Search Console, a Google service account with appropriate permissions is required.
- For the link-building marketplace, user marketplace credentials must be configured.
- The `sitemaps` action requires a `payload.sitemaps` array with valid sitemap URLs.
