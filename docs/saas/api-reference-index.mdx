# Which API do I need?

1Platform Labs exposes two independent SaaS APIs. Pick the one that matches what you're building, then
open its interactive reference to browse and test endpoints live.

| API | Build… | Base URL | Reference |
|---|---|---|---|
| **1Platform API** | AI content, payments, invoicing, domains, agents — the full product platform. | `https://api.1platform.pro/api/v1` | [Open →](/api-reference/1platform-api) |
| **Atlas API** | A multitenant content-delivery ecosystem — catalog, entitlements, delivery, webhooks. | `https://atlas-api.1platform.pro/api/v1` | [Open →](/api-reference/atlas-api) |

## 1Platform API

The core platform API. **Two-token** authentication (`Authorization: Bearer ak-...` +
`x-user-token: sk-...`). Start with the [overview](/docs/saas/1platform-api/overview) and
[getting started](/docs/saas/1platform-api/getting-started).

- [Interactive reference →](/api-reference/1platform-api)

## Atlas API

A standalone, multitenant content-delivery platform. **JWT bearer** authentication plus per-tenant
resolution; independent from the 1Platform API. Start with the
[overview](/docs/saas/atlas-api/overview).

- [Interactive reference →](/api-reference/atlas-api)

:::note
The interactive references render each API's OpenAPI specification. Their content stays in the API's
source language; the surrounding portal is available in English and Spanish.
:::
