# Webhook Security

Every outbound webhook is signed with HMAC-SHA256 using the shared
`webhook_secret` returned at domain-registration time (or rotation time). Verify
the signature **before** trusting the payload.

## Signing scheme

| Property | Value |
|---|---|
| Signature header | `X-Webhook-Signature` |
| Signature format | `sha256=<hex>` |
| Companion header | `X-Webhook-Timestamp` (unix epoch seconds, string) |
| Algorithm | `HMAC-SHA256` |
| Signed material | `f"{timestamp}.".encode() + payload_bytes` |
| Body canonicalisation (sender) | `json.dumps(body, sort_keys=True, separators=(",",":"))` |
| Replay window | 5 minutes (`X-Webhook-Timestamp` skew) |

:::warning Verify against the raw bytes you receive
1Platform signs the canonical re-serialisation of the body, but you **must
verify against the raw bytes that arrived on the wire** — not the result of
re-stringifying a parsed JSON object. Any whitespace difference will break the
HMAC.
:::

## Verification — Python (httpx + hmac)

```python
import hmac
import hashlib
import time

REPLAY_WINDOW_SECONDS = 300  # 5 minutes

def verify_webhook(
    *,
    body: bytes,                   # raw request body
    signature_header: str,         # value of X-Webhook-Signature
    timestamp_header: str,         # value of X-Webhook-Timestamp
    secret: str,                   # your webhook_secret
) -> bool:
    if not signature_header.startswith("sha256="):
        return False

    # Reject anything outside the replay window
    try:
        ts = int(timestamp_header)
    except ValueError:
        return False
    if abs(time.time() - ts) > REPLAY_WINDOW_SECONDS:
        return False

    expected = signature_header.removeprefix("sha256=")
    signed = f"{timestamp_header}.".encode() + body
    actual = hmac.new(secret.encode(), signed, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, actual)
```

## Verification — TypeScript (Node 22+)

```ts
import { createHmac, timingSafeEqual } from "node:crypto";

const REPLAY_WINDOW_SECONDS = 300;

export function verifyWebhook(opts: {
  body: Buffer;
  signatureHeader: string;
  timestampHeader: string;
  secret: string;
}): boolean {
  const { body, signatureHeader, timestampHeader, secret } = opts;
  if (!signatureHeader.startsWith("sha256=")) return false;

  const ts = Number.parseInt(timestampHeader, 10);
  if (!Number.isFinite(ts)) return false;
  if (Math.abs(Date.now() / 1000 - ts) > REPLAY_WINDOW_SECONDS) return false;

  const expected = signatureHeader.slice("sha256=".length);
  const signed = Buffer.concat([Buffer.from(`${timestampHeader}.`), body]);
  const actual = createHmac("sha256", secret).update(signed).digest("hex");

  return (
    expected.length === actual.length &&
    timingSafeEqual(Buffer.from(expected), Buffer.from(actual))
  );
}
```

## Verification — PHP

```php
<?php
const REPLAY_WINDOW_SECONDS = 300;

function verifyWebhook(string $rawBody, string $signatureHeader, string $timestampHeader, string $secret): bool {
    if (strpos($signatureHeader, 'sha256=') !== 0) {
        return false;
    }
    $ts = (int) $timestampHeader;
    if (abs(time() - $ts) > REPLAY_WINDOW_SECONDS) {
        return false;
    }

    $expected = substr($signatureHeader, strlen('sha256='));
    $signed = $timestampHeader . '.' . $rawBody;
    $actual = hash_hmac('sha256', $signed, $secret);

    return hash_equals($expected, $actual);
}
```

## Common failure modes

- **Re-stringifying the body before verifying** — see the warning above.
- **Trimming the `sha256=` prefix in the wrong place** — strip it from
  `signature_header`, not from the computed value.
- **Missing replay window** — without a timestamp check an attacker can
  replay a captured payload indefinitely.
- **String comparison instead of constant-time comparison** — use
  `hmac.compare_digest`, `timingSafeEqual`, `hash_equals`.

## Secret rotation

:::danger The webhook secret is shown only once
`POST /webhooks/config/rotate-secret` returns the new `webhook_secret` exactly
once. There is no `GET` endpoint. If you lose the response, you must rotate
again — and any in-flight webhooks signed with the previous secret will fail
verification at your end.

**Always pipe the response to a secret-manager write before any other action.**
:::

A safe rotation runbook:

1. Call `POST /webhooks/config/rotate-secret` in a script that captures `stdout`
   directly to your secret manager (e.g. `vault kv put`, `gcloud secrets`).
2. Keep the previous secret in your verifier for **24 hours** (accept either)
   to absorb in-flight retries from before the rotation.
3. After 24 h, remove the old secret from the verifier.
