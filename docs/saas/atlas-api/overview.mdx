# Atlas API

The **Atlas API** is a standalone, multitenant content-delivery platform by 1Platform Labs. Use it to
build your own branded ecosystem — a library, streaming service, podcast network, or publishing
platform — with catalog management, authentication, entitlements, and media delivery, all isolated
per tenant.

> Atlas is **independent** from the 1Platform API: different auth model, its own data, no shared
> internals. For the core platform API, see [1Platform API](/docs/saas/1platform-api/overview).

## How it differs from the 1Platform API

| | 1Platform API | Atlas API |
|---|---|---|
| **Auth** | Two-token (`ak-` + `sk-`) | **JWT bearer** (after magic-link) + integration keys |
| **Tenant** | Per `app` / `user` | Resolved by `Host` / `X-Tenant-Slug` |
| **Error shape** | `{ success, data, msg }` | `{ code, message }` |
| **Base URL** | `api.1platform.pro/api/v1` | `atlas-api.1platform.pro/api/v1` |

## Core integration flows

- [Authentication](/docs/saas/atlas-api/authentication) — magic-link → JWT, refresh, sessions.
- [Integration keys](/docs/saas/atlas-api/integration-keys) — service-to-service access.
- [Tenant resolution](/docs/saas/atlas-api/tenant-resolution) — resolve a tenant by host or slug.
- [Catalog browsing](/docs/saas/atlas-api/catalog-browsing) — list, fetch by slug, search.
- [Content delivery](/docs/saas/atlas-api/content-delivery) — playback tickets, signed URLs, streaming.
- [Entitlements](/docs/saas/atlas-api/entitlements) — grant, revoke, and check access (scopes: item, series, plan, live).
- [Webhooks](/docs/saas/atlas-api/webhooks) — receive entitlement events (signed, idempotent).
