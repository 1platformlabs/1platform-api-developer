# 1Platform Dashboard

The **1Platform Dashboard** is the white-label control panel your workspace uses to operate
everything 1Platform powers — billing and credits, invoicing, custom domains, team and roles,
API keys, and a modular feature set that you turn on or off per tenant.

Every tenant runs the same product but gets its **own brand**: logo, colors, typography, name,
enabled modules, and feature flags are resolved at runtime, so the dashboard looks and behaves like
*your* product, not a generic tool.

> This section is for **operators** configuring the product for a workspace — no code required.
> If you want to integrate programmatically, see the [1Platform API](/docs/saas/1platform-api/overview).

## What you can do

| Area | What an operator does |
|---|---|
| **Dashboard** | See workspace stats at a glance — balance, usage this month, recent activity. |
| **Billing & credits** | Check balance, top up credits, review the full transaction history. |
| **Invoicing** | Issue and track invoices by status, and manage the issuing businesses. |
| **Domains** | Add and manage the custom domains for your workspace. |
| **Team & roles** | Invite members, assign roles, deactivate accounts. |
| **API keys** | Generate and rotate the keys integrations use. |
| **Settings** | Profile, workspace, security, and notification preferences. |
| **Modules** | Enable optional modules — websites, tools, plugins, tasks, tickets, FAQs, referrals. |

## White-label by design

What you can customize per tenant:

- **Brand** — name, logo (light/dark), favicon, social image, support email, legal links.
- **Appearance** — color palette, typography, light surfaces, component styling.
- **Modules** — which features appear in the sidebar, their order, and who can access them.
- **Feature flags** — progressive rollout of capabilities per workspace.

Sign-in is **magic-link only** — members receive a one-time link by email; there is no password to
manage.
