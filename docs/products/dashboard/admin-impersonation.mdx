# Administración e impersonación

Algunos controles están reservados para los **administradores del workspace**. Si no tienes permisos de administrador, estas pantallas muestran un mensaje pidiéndote que contactes a un administrador para obtener acceso.

## Configuración de la app

La pantalla de administración **Configuración de la app** gestiona la configuración a nivel de workspace:

- **Nombre visible** — el nombre visible del workspace. Edítalo y selecciona **Guardar**.
- **Integraciones** — un resumen de cuántas integraciones están configuradas (por ejemplo, "2 de 3 configuradas"). Las credenciales específicas de cada integración se gestionan con soporte; el dashboard solo muestra el conteo.
- **Clave API del workspace** — rota la clave que usan las integraciones.

## Rotar la clave API del workspace

1. Selecciona **Rotar clave API**.
2. Confirma — la acción es inmediata e irreversible.
3. La nueva clave se muestra **una sola vez**. Cópiala de inmediato.

La rotación invalida cada token de integración activo, así que actualiza tus integraciones y cualquier herramienta conectada inmediatamente después. Consulta [Claves API](/docs/products/dashboard/api-keys).

## Ver como un miembro (impersonación)

Desde la pantalla de administración **Usuarios**, un administrador puede elegir **Ver como** junto a un miembro para entrar a la vista que ese miembro tiene del workspace. Esto es útil para diagnosticar qué ve una persona específica. El dashboard se recarga bajo la identidad del miembro, y el administrador puede salir para volver a su propia sesión después. Consulta [Equipo y roles](/docs/products/dashboard/team-and-roles).

## Permisos

Estas pantallas requieren el rol de administrador del workspace. Un miembro que no lo tenga ve un mensaje claro de "no tienes permisos de administrador" en lugar de un error — solicita acceso a un administrador del workspace si lo necesitas.
