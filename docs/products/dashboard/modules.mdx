# Modules

The dashboard is **modular**. Each capability is a module that can be enabled or disabled per workspace, so two workspaces running the same product can offer very different feature sets.

## How modules appear

Enabled modules show up as entries in the left sidebar. For each module, the workspace configuration controls:

- **Whether it's enabled** — turned on or off for the workspace.
- **Its label and icon** — how it reads in the sidebar.
- **Its order** — where it sits in the list.
- **Who can see it** — permission requirements, and whether it's admin-only.

Some modules also have **sub-items** that appear nested beneath them.

## Commonly enabled modules

| Module | What it does |
|---|---|
| **Dashboard** | Workspace stats, usage trend, and activity. |
| **Billing** | Balance, top-ups, and transactions. |
| **Invoicing** | Issue and track invoices; manage issuing businesses. |
| **Domains** | Connect and monitor custom domains. |
| **Onboarding** | The guided setup checklist. |
| **Settings** | Profile, workspace, security, and more. |
| **Notifications** | Workspace alerts. |
| **Logs** (admin) | API request history. |

## Optional modules

Additional modules can be switched on when a workspace needs them — for example websites, tools, plugins, referrals, tasks, tickets, and FAQs. They stay hidden until enabled, keeping each workspace's sidebar focused on what it actually uses.

## Feature flags

Alongside modules, **feature flags** let capabilities be rolled out gradually to specific workspaces. This is how new functionality can be tried by some workspaces before it's turned on everywhere.

## Configuring modules

Modules and feature flags are part of the per-workspace configuration. To enable or reorder a module, contact your workspace administrator or support. Appearance and brand are configured the same way — see [Branding & appearance](/docs/products/dashboard/branding-appearance).
