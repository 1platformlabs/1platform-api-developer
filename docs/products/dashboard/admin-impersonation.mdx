# Admin & impersonation

Some controls are reserved for **workspace administrators**. If you don't have admin permissions, these screens show a message asking you to contact an administrator for access.

## App settings

The admin **App settings** screen manages workspace-level configuration:

- **Display name** — the workspace's display name. Edit it and select **Save**.
- **Integrations** — a summary of how many integrations are configured (for example, "2 of 3 configured"). Specific integration credentials are managed with support; the dashboard only shows the count.
- **Workspace API key** — rotate the key that integrations use.

## Rotating the workspace API key

1. Select **Rotate API key**.
2. Confirm — the action is immediate and irreversible.
3. The new key is shown **once**. Copy it right away.

Rotating invalidates every active integration token, so update your integrations and any connected tools immediately afterward. See [API keys](/docs/products/dashboard/api-keys).

## Viewing as a member (impersonation)

From the admin **Users** screen, an administrator can choose **View as** next to a member to enter that member's view of the workspace. This is useful for troubleshooting what a specific person sees. The dashboard reloads under the member's identity, and the admin can exit to return to their own session afterward. See [Team & roles](/docs/products/dashboard/team-and-roles).

## Permissions

These screens require the workspace admin role. A member without it sees a clear "you don't have admin permissions" message rather than an error — request access from a workspace administrator if you need it.
