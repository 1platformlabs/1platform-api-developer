# Quickstart

This walkthrough takes you from zero to a working **Atlas API** integration: pick an environment,
resolve a tenant, sign a member in with a magic link, browse the public catalog, and mint a playback
ticket for an entitled item.

> Atlas is multitenant. **Every** request is scoped to a tenant resolved server-side from the
> `Host` header (or an explicit `X-Tenant-Slug` override) — see
> [Tenant resolution](/docs/saas/atlas-api/tenant-resolution). You never put a tenant id in the body.

## 1. Set your base URL

```bash
# Production
export ATLAS_BASE="https://atlas-api.1platform.pro/api/v1"

# QA / staging
# export ATLAS_BASE="https://atlas-api-qa.1platform.pro/api/v1"

# The tenant you are integrating against, by slug or by its primary domain:
export TENANT_SLUG="your-tenant"
```

See [Environments](/docs/saas/atlas-api/environments) for the full list.

## 2. Resolve the tenant (optional sanity check)

The public branding endpoint confirms your tenant resolves and returns its archetype and config:

```bash
curl -s "$ATLAS_BASE/tenants/by-host?host=library.example.com"
```

```json
{
  "id": "9d1c…",
  "slug": "your-tenant",
  "name": "Example Library",
  "archetype": "library",
  "primary_domain": "library.example.com",
  "config": { "appearance": "light" },
  "is_active": true
}
```

## 3. Sign a member in (magic link → JWT)

Atlas authenticates members with a **passwordless magic link**. Request the link, the member clicks
it (delivering a single-use token), then exchange that token for a JWT session.

```bash
# (a) Issue the magic link — the member receives an email
curl -s -X POST "$ATLAS_BASE/auth/magic-link" \
  -H "X-Tenant-Slug: $TENANT_SLUG" \
  -H "Content-Type: application/json" \
  -d '{ "email": "reader@example.com", "return_url": "/me/library" }'

# (b) Exchange the single-use token (from the email link) for a session
curl -s -X POST "$ATLAS_BASE/auth/magic-link/verify" \
  -H "X-Tenant-Slug: $TENANT_SLUG" \
  -H "Content-Type: application/json" \
  -d '{ "token": "<TOKEN_FROM_EMAIL_LINK>" }'
```

```json
{
  "member_id": "1f2e…",
  "access_token": "<JWT_ACCESS_TOKEN>",
  "refresh_token": "<JWT_REFRESH_TOKEN>",
  "expires_in": 86400,
  "member": { "id": "1f2e…", "email": "reader@example.com", "tenant_id": "9d1c…" },
  "roles": ["fan"]
}
```

Save the access token and send it on subsequent calls:

```bash
export ATLAS_JWT="<JWT_ACCESS_TOKEN>"
```

Full details (refresh, sessions, admin sign-in) in [Authentication](/docs/saas/atlas-api/authentication).

## 4. Browse the catalog

The catalog is public (anonymous) and tenant-scoped:

```bash
curl -s "$ATLAS_BASE/public/catalog?page=1&page_size=20" \
  -H "X-Tenant-Slug: $TENANT_SLUG"
```

```json
{
  "items": [
    {
      "id": "a3b4…",
      "slug": "the-time-machine",
      "kind": "book",
      "title": "The Time Machine",
      "status": "published",
      "cover_image_url": "https://…/covers/the-time-machine.jpg",
      "coming_soon": false,
      "metadata": { "author": "H. G. Wells" }
    }
  ],
  "total": 502,
  "page": 1,
  "page_size": 20
}
```

More query patterns in [Catalog browsing](/docs/saas/atlas-api/catalog-browsing).

## 5. Mint a playback ticket

With a JWT, request a short-lived **playback ticket** for an item. Atlas checks the member's
entitlement first, then returns a signed URL (or an HLS envelope, depending on the tenant archetype):

```bash
curl -s "$ATLAS_BASE/delivery/items/<CONTENT_ID>/playback?format=epub" \
  -H "X-Tenant-Slug: $TENANT_SLUG" \
  -H "Authorization: Bearer $ATLAS_JWT"
```

```json
{
  "content_id": "a3b4…",
  "delivery_kind": "signed_url",
  "url": "https://storage.example/…?signature=…&expires=…",
  "expires_in_seconds": 600,
  "drm_token": null,
  "renderable": true,
  "render_format": "epub"
}
```

A member without access gets `403 { "code": "ENTITLEMENT_DENIED" }`. See
[Entitlements](/docs/saas/atlas-api/entitlements) and [Content delivery](/docs/saas/atlas-api/content-delivery).
