# Atlas Dashboard

The **Atlas Dashboard** is the white-label web app operators use to run a branded digital
distribution platform — a public storefront (library, shows, live channels, purchases) plus an admin
panel for content, monetization, members, SEO, analytics, and the companion mobile app.

Each tenant brings its **own domain** (via a simple DNS record), and the platform resolves branding,
copy, and configuration automatically from the host — so your storefront is fully *yours*.

> This section is for **operators** running a tenant — no code required. To build on the platform
> programmatically, see the [Atlas API](/docs/saas/atlas-api/overview).

## What you can do

| Area | What an operator does |
|---|---|
| **Content** | Upload, edit, schedule and organize items (books, video, audio, live). |
| **Catalog & taxonomy** | Authors, publishers, genres, and cross-links between kinds. |
| **Storefront pages** | About pages, contact, legal — editorial content for your site. |
| **Live channels** | Schedule broadcasts and manage the program guide. |
| **Monetization** | Subscription tiers, purchases, rentals, and ads (when enabled). |
| **Members** | Directory, roles, moderation, and parental controls. |
| **Mobile app** | Store links, app analytics, and app-exclusive content. |
| **SEO** | Metadata, verification, sitemaps, crawl health, redirects. |
| **Branding** | Theme, typography, colors, logo, footer, and copy. |
| **Analytics & audit** | Usage insight and an admin activity log. |

## White-label by design

Configured per tenant and resolved at runtime from the host:

- **Appearance** — theme (light/dark), typographic voice, cover style, primary & accent colors.
- **Branding** — logo, favicon, hero, site name, tagline, organization.
- **SEO & meta** — title templates, social cards, sitemaps, canonical host, verification.
- **Footer & social** — links, social profiles, copyright.
- **URL routing** — kind-to-path mapping and legacy redirects.
