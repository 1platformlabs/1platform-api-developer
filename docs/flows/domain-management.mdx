# Flow: Domain Management

## Overview

This flow documents how to manage domains through the 1Platform API. It covers checking domain availability, retrieving TLD pricing, registering domains, managing DNS records and nameservers, initiating transfers, renewing domains, and tracking operations history.

**Use case:** Provide your users with a complete domain management experience -- from searching and registering domains to configuring DNS records and handling transfers -- all through a single API.

**End result:** A fully registered domain with configured DNS records, WHOIS privacy, and the ability to manage nameservers, renewals, locks, and transfers programmatically.

---

## Prerequisites

| Requirement | Details |
|---|---|
| App API Key | Application key (`ak-...`) provided when registering your app |
| User API Key | User key (`sk-...`) obtained when creating the user |
| Sufficient balance | Domain registration and renewal require balance in the user's account |
| Domain contact | A contact profile must be configured before registering domains |

### Required headers for all requests

```
Authorization: Bearer $APP_TOKEN
x-user-token: $USER_TOKEN
Content-Type: application/json
```

## Data required

### Contact

| Field | Type | Required | Description |
|---|---|---|---|
| `first_name` | string | Yes | Contact first name (max 100 chars) |
| `last_name` | string | Yes | Contact last name (max 100 chars) |
| `email` | string | Yes | Contact email (max 255 chars, email format) |
| `phone` | string | Yes | Phone in E.164 dot format: `+N.NNNNNNN` (max 20 chars) |
| `address` | string | Yes | Street address (max 255 chars) |
| `city` | string | Yes | City (max 100 chars) |
| `state` | string | Yes | State or province (max 100 chars) |
| `country` | string | Yes | ISO 3166-1 alpha-2 country code (exactly 2 chars) |
| `postal_code` | string | Yes | Postal/ZIP code (max 20 chars) |
| `organization` | string or null | No | Organization name (max 255 chars) |

### Registration

| Field | Type | Required | Description |
|---|---|---|---|
| `domain` | string | Yes | Domain name to register, e.g. `"example.com"` (max 253 chars) |
| `years` | integer | No | Registration period, 1-10 (default: 1) |
| `whois_privacy` | boolean | No | Enable WHOIS privacy (default: true) |
| `auto_renew` | boolean | No | Enable automatic renewal (default: false) |
| `nameservers` | array of strings or null | No | Custom nameservers (max 12) |

### DNS Records

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | string | Yes | Record type: `A`, `AAAA`, `CNAME`, `MX`, `TXT`, `NS`, `URL_REDIRECT`, `URL_FRAME` |
| `host` | string | Yes | Hostname, e.g. `"@"`, `"www"`, `"mail"` (max 253 chars) |
| `value` | string | Yes | Record value: IP, hostname, text, etc. (max 1024 chars) |
| `ttl` | integer | No | Time to live in seconds, 60-86400 (default: 1800) |
| `mx_pref` | integer or null | No | MX priority, 0-65535 (required for MX records) |

### Transfer

| Field | Type | Required | Description |
|---|---|---|---|
| `domain` | string | Yes | Domain to transfer (max 253 chars) |
| `epp_code` | string | Yes | EPP/auth code from current registrar (max 255 chars) |
| `years` | integer | No | Years to add on transfer, 1-10 (default: 1) |
| `whois_privacy` | boolean | No | Enable WHOIS privacy (default: true) |
| `auto_renew` | boolean | No | Enable automatic renewal (default: false) |

## Step-by-step flow

### Step 0 — Get App Token

**Goal:** Authenticate the application and obtain the app JWT token.

**Endpoint:** `POST /api/v1/auth/token`

**Auth:** None (this is the entry point)

```bash
curl -X POST "$BASE_URL/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$APP_API_KEY'"}'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "App token generated successfully"
}
```

:::note
Save `data.access_token` as `$APP_TOKEN`. Expires in 30 minutes.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Invalid API key or inactive app | Verify that `apiKey` is correct and the app is active |
| 422 | Missing or malformed `apiKey` field | Send `{"apiKey": "..."}` in the body |
| 429 | Rate limit exceeded (10 req/min) | Wait and retry |

### Step 2 — Create or Update Domain Contact

**Goal:** Configure the contact profile used for domain registration. This must be done before registering any domain.

**Endpoint:** `PUT /api/v1/domains/contact`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X PUT "$BASE_URL/domains/contact" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Carlos",
    "last_name": "Ejemplo Demo",
    "email": "carlos@example.com",
    "phone": "+502.00001234",
    "address": "Avenida Reforma 10-50",
    "city": "Guatemala",
    "state": "Guatemala",
    "country": "GT",
    "postal_code": "01010",
    "organization": null
  }'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "first_name": "Carlos",
    "last_name": "Ejemplo Demo",
    "email": "carlos@example.com",
    "phone": "+502.00001234",
    "address": "Avenida Reforma 10-50",
    "city": "Guatemala",
    "state": "Guatemala",
    "country": "GT",
    "postal_code": "01010",
    "organization": null,
    "updated_at": "2026-03-18T10:00:00"
  },
  "msg": "Contact saved successfully"
}
```

:::note
This endpoint is idempotent -- calling it multiple times updates the same contact profile. You only need one contact per user.
:::

:::warning
A contact profile is **required** before registering any domain (Step 5) or initiating a transfer (Step 13). If the contact is not configured, those endpoints return a **404** error.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | Invalid contact data (validation failed) | Verify all required fields and formats (e.g., `country` must be 2 chars, `phone` must match `+N.NNNNNNN`) |
| 401 | Authentication required | Regenerate tokens |
| 422 | Missing or malformed fields | Verify all required fields are present and valid |
| 429 | Rate limit exceeded | Wait and retry |

### Step 3 — Check Domain Availability

**Goal:** Check whether one or more domains are available for registration.

**Endpoint:** `GET /api/v1/domains/check`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X GET "$BASE_URL/domains/check?domain=example-demo.com&domain=tienda-ejemplo.com" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN"
```

**Response (200):**

```json
{
  "success": true,
  "data": [
    {
      "domain": "example-demo.com",
      "available": true,
      "price": 12.99,
      "currency": "USD",
      "period": 1
    },
    {
      "domain": "tienda-ejemplo.com",
      "available": false,
      "price": null,
      "currency": null,
      "period": null
    }
  ],
  "msg": "Domain availability checked"
}
```

:::tip
You can check up to **50 domains** in a single request by repeating the `domain` query parameter.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | Invalid domain format or too many domains (max 50) | Verify domain format and reduce the number of domains |
| 401 | Authentication required | Regenerate tokens |
| 422 | Missing or malformed `domain` parameter | Provide at least one `domain` query parameter |
| 429 | Rate limit exceeded | Wait and retry |
| 502 | Domain provider temporarily unavailable | Retry after a few seconds |
| 503 | Domain service disabled | Contact support -- the domain service may be under maintenance |

### Step 5 — Register a Domain

**Goal:** Register a new domain for the authenticated user.

**Endpoint:** `POST /api/v1/domains`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/domains" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "example-demo.com",
    "years": 1,
    "whois_privacy": true,
    "auto_renew": false,
    "nameservers": null
  }'
```

**Response (201):**

```json
{
  "success": true,
  "data": {
    "id": "507f1f77bcf86cd799439011",
    "domain": "example-demo.com",
    "status": "active",
    "expires_at": "2027-03-18T10:30:00",
    "created_at": "2026-03-18T10:30:00",
    "auto_renew": false,
    "whois_privacy": true,
    "locked": true,
    "nameservers": [
      "ns1.default-dns.com",
      "ns2.default-dns.com"
    ]
  },
  "msg": "Domain registered successfully"
}
```

:::note
Save `data.id` as `$DOMAIN_ID` for subsequent requests.
:::

:::warning
Domain registration charges the user's balance immediately. Verify availability (Step 3) and sufficient balance before calling this endpoint.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | Invalid domain name | Verify domain format (e.g., `example.com`, max 253 chars) |
| 401 | Authentication required | Regenerate tokens |
| 402 | Insufficient balance | Add funds to the user's account before retrying |
| 404 | Contact not configured | Create a domain contact first (Step 2) |
| 409 | Domain not available or already registered | Check availability first (Step 3), or the domain is already in the user's account |
| 422 | Validation error in request body | Check field formats and constraints |
| 429 | Rate limit exceeded or domain limit reached | Wait and retry, or check account domain limits |
| 502 | Domain provider failed to process | Retry after a few seconds |
| 503 | Domain service disabled | Contact support |

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X PUT "$BASE_URL/domains/$DOMAIN_ID/dns" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "records": [
      {
        "type": "A",
        "host": "@",
        "value": "93.184.216.34",
        "ttl": 3600

      {
        "type": "CNAME",

      }
```

```json
  "data": {
    "records": [

        "type": "A",
        "host": "@",
        "value": "93.184.216.34",

        "mx_pref": null

        "type": "CNAME",
```

This endpoint **replaces all existing records**. Always include all desired records in the request, not just the ones you want to add or change. Retrieve current records first (Step 8) to avoid accidentally removing existing records.

MX records require the `mx_pref` field. For other record types, `mx_pref` can be omitted or set to `null`.

### Step 10 — Set Domain Nameservers

**Goal:** Switch between default and custom nameservers for a domain.

**Endpoint:** `PUT /api/v1/domains/{domain_id}/nameservers`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X PUT "$BASE_URL/domains/$DOMAIN_ID/nameservers" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "custom",
    "nameservers": [
      "ns1.cloudflare.com",
      "ns2.cloudflare.com"
    ]
  }'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "type": "custom",
    "nameservers": [
      "ns1.cloudflare.com",
      "ns2.cloudflare.com"
    ]
  },
  "msg": "Nameservers updated"
}
```

:::note
When `type` is `"default"`, the `nameservers` field is ignored and the domain reverts to the provider's default nameservers. When `type` is `"custom"`, the `nameservers` array is required.
:::

:::warning
Changing nameservers to custom values means DNS records managed through this API (Steps 8-9) will no longer apply. DNS will be resolved by the custom nameserver provider instead.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | Invalid nameserver format | Verify nameserver hostnames are valid FQDNs |
| 401 | Authentication required | Regenerate tokens |
| 404 | Domain not found | Verify the `domain_id` |
| 422 | Validation error | Verify `type` is `"default"` or `"custom"`, and `nameservers` array has max 12 items |
| 429 | Rate limit exceeded | Wait and retry |
| 502 | Provider failed to update nameservers | Retry after a few seconds |

**Common errors:**

| Code | Cause | Solution |

### Step 12 — Set Registrar Lock

**Goal:** Lock or unlock a domain to prevent unauthorized transfers.

**Endpoint:** `PATCH /api/v1/domains/{domain_id}/lock`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X PATCH "$BASE_URL/domains/$DOMAIN_ID/lock" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "locked": false
  }'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "id": "507f1f77bcf86cd799439011",
    "domain": "example-demo.com",
    "locked": false
  },
  "msg": "Domain lock updated"
}
```

:::warning
Unlocking a domain is required before initiating an outbound transfer. Always re-lock the domain after a transfer is cancelled or completed to prevent unauthorized transfers.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Authentication required | Regenerate tokens |

## Idempotency and best practices

### Idempotency

:::warning[No idempotency mechanism]
The API does not support an `Idempotency-Key` header. If a domain registration request fails due to timeout, list the user's domains (`GET /domains`) before retrying to avoid duplicate charges. The same domain cannot be registered twice -- the API returns a **409** error.
:::

### Retries
- If you receive **429** (rate limit), wait the indicated time and retry.
- If you receive **500** or a **timeout**, verify the resource state before retrying.
- For **401** (expired token), regenerate the token and retry the original request.
- For **502** (provider unavailable), wait a few seconds and retry -- the external provider may be temporarily overloaded.

### Timeouts
- Domain availability checks typically complete within **2-5 seconds**.
- Domain registration may take **5-15 seconds** depending on the TLD and provider.
- DNS record updates typically complete within **2-5 seconds**.
- Domain transfers are asynchronous and take **5-7 days** -- poll the transfer status endpoint periodically.
- Configure your HTTP client with at least **30 seconds** timeout for registration and renewal endpoints.

### Tokens
- Both tokens (app and user) expire in **30 minutes** (1800 seconds). Implement automatic renewal before expiration.
- Both tokens are required on all domain management endpoints.

### Important validations
- A domain contact **must** be configured before registering domains or initiating transfers.
- Domain names must be valid FQDNs (max 253 characters).
- Phone numbers must use the E.164 dot format: `+N.NNNNNNN` (e.g., `+502.00001234`).
- Country codes must be exactly 2 characters (ISO 3166-1 alpha-2).
- DNS records are replaced entirely on each `PUT` -- always include all desired records.
- MX records require the `mx_pref` field.
- Nameserver changes take effect within minutes but may take up to 48 hours to propagate globally.
- Domain registration and renewal charges are non-refundable.
