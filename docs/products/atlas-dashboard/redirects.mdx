# Redirects

The **Redirects** screen (`/admin/redirects`) lets you forward old URLs to new ones. This protects
existing links and search ranking when you rename pages, migrate from a previous site, or restructure
your catalog. It works hand in hand with
[SEO Configuration](/docs/products/atlas-dashboard/seo-configuration).

## Rule-based redirects

The **static rules** table matches URLs by pattern, so one rule can cover many pages:

| Column | Purpose |
|---|---|
| From pattern | The URL pattern to match |
| To template | The destination (can reuse matched parts) |
| Status | The redirect type (e.g. permanent or temporary) |

A blank row is always available to add a new rule.

## Bulk upload

The **bulk upload** area accepts a simple list of rows — old path, new path, and status — so you can
import many redirects at once, for example when migrating from an older site.

## Per-URL redirects

The **per-row** table lists individual legacy paths mapped to their target, with a status code and a
delete button per row. Use this for one-off URL moves that don't fit a pattern.

## Tips

- Prefer permanent redirects for moves you intend to keep.
- Keep redirects pointing directly at the final URL rather than chaining several together.
- When you rename item URLs, legacy paths are kept working for a grace period automatically, so most
  catalog renames don't require manual redirects.
