# Activity & request logs

The **API request logs** screen (admin) shows every request made to your workspace. It's a useful place to confirm that integrations are working and to spot anything failing.

## The log table

Each row records a single request:

| Column | What it shows |
|---|---|
| **Time** | When the request arrived. |
| **Method** | The kind of request (read, create, update, delete). |
| **Endpoint** | Which part of the API was called. |
| **Status** | The result code — green for success, amber for client issues, red for server errors. |
| **Duration** | How long the request took, in milliseconds. |

Status codes are color-coded so you can scan for problems at a glance.

## Filtering

Narrow the list with the controls above the table:

- **Search** — by endpoint or method.
- **Method** — all, or a single request type.
- **Time range** — today, last 7 days, or last 30 days.

## Exporting

Use **Export** to download the log for record-keeping or deeper analysis.

## Empty and loading states

If the workspace hasn't received traffic yet, an empty state explains that entries will appear as soon as requests come in. Placeholders show briefly while the log loads.

## Where it connects

The dashboard home's **Recent activity** feed is a lighter view of the same events; choosing **View all** there brings you to this full log. See [Dashboard home](/docs/products/dashboard/dashboard-home).
