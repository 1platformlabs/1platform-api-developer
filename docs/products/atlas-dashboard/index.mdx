# Atlas Dashboard

El **Atlas Dashboard** es la app web white-label que usan los operadores para administrar una plataforma de
distribución digital con marca propia: un storefront público (biblioteca, shows, canales en vivo, compras) más un panel de
administración para contenido, monetización, miembros, SEO, analíticas y la app móvil complementaria.

Cada tenant trae su **propio dominio** (mediante un registro DNS simple), y la plataforma resuelve la marca,
los textos y la configuración automáticamente desde el host, para que tu storefront sea totalmente *tuyo*.

> Esta sección es para **operadores** que administran un tenant: no se requiere código. Para construir sobre la plataforma
> de forma programática, consulta la [Atlas API](/docs/saas/atlas-api/overview).

## Qué puedes hacer

| Área | Qué hace un operador |
|---|---|
| **Contenido** | Sube, edita, programa y organiza ítems (libros, video, audio, en vivo). |
| **Catálogo y taxonomía** | Autores, editoriales, géneros y enlaces cruzados entre tipos. |
| **Páginas del storefront** | Páginas Acerca de, contacto, legal: contenido editorial para tu sitio. |
| **Canales en vivo** | Programa transmisiones y gestiona la guía de programación. |
| **Monetización** | Niveles de suscripción, compras, alquileres y anuncios (cuando están habilitados). |
| **Miembros** | Directorio, roles, moderación y controles parentales. |
| **App móvil** | Enlaces de tienda, analíticas de la app y contenido exclusivo de la app. |
| **SEO** | Metadatos, verificación, sitemaps, salud de rastreo, redirecciones. |
| **Marca** | Tema, tipografía, colores, logo, pie de página y textos. |
| **Analíticas y auditoría** | Información de uso y un registro de actividad de administración. |

## White-label por diseño

Configurado por tenant y resuelto en tiempo de ejecución desde el host:

- **Apariencia**: tema (claro/oscuro), voz tipográfica, estilo de portada, colores primario y de acento.
- **Marca**: logo, favicon, hero, nombre del sitio, lema, organización.
- **SEO y meta**: plantillas de título, tarjetas sociales, sitemaps, host canónico, verificación.
- **Pie de página y redes**: enlaces, perfiles sociales, derechos de autor.
- **Enrutamiento de URL**: mapeo de tipo a ruta y redirecciones heredadas.
