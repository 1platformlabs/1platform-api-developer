# Límites de uso

Los límites de uso se aplican mediante [slowapi](https://slowapi.readthedocs.io/) en
cada endpoint protegido. Superar el límite devuelve `429 Too Many Requests`.

## Clave de límite de uso

La clave de límite de uso es el SHA-256 de `(header Authorization + header x-user-token)`.
Cuando ambos headers están ausentes, la API recurre a la dirección IP del cliente.

## Límites por endpoint

| Constante | Valor | Endpoints |
|---|---|---|
| `HEALTH_LIMIT` | 100 / min | `/health` |
| `AUTH_LIMIT` | 10 / min | `POST /auth/token`, `POST /users/token` |
| `READ_LIMIT` | 50 / min | Todos los endpoints `GET` |
| `WRITE_LIMIT` | 20 / min | Todos los `POST`/`PUT`/`PATCH`/`DELETE` (excepto los especiales de abajo) |
| `KEYWORD_LIMIT` | 10 / min | `/posts/keywords/*` |
| `CONTENT_LIMIT` | 5 / min | `/posts/content/*` |
| `INDEXING_LIMIT` | 20 / min | `/posts/indexing/*` |

Los límites son idénticos en QA y producción.

## Cómo manejar 429

```python
import time
import httpx

def call_with_backoff(client: httpx.Client, *args, **kwargs) -> httpx.Response:
    for attempt in range(3):
        r = client.request(*args, **kwargs)
        if r.status_code != 429:
            return r
        retry_after = int(r.headers.get("Retry-After", "60"))
        time.sleep(retry_after)
    return r  # exhausted retries
```

:::tip[Guarda los tokens en caché para no superar AUTH_LIMIT]
El límite de 10 / min en `auth/token` y `users/token` es la razón más común
para alcanzar un 429 durante el desarrollo. Guarda los tokens en caché por `expires_in - 60s` y solo
genera uno nuevo cuando el valor en caché esté por expirar. Consulta el
[patrón de caché de tokens](../flows/payments-and-subscriptions#token-caching-pattern).
:::
