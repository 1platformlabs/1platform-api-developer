# Content Management

The **Content** screen (`/admin/content`) is your catalog inventory. The header shows your total item
count and how many were added this week, and a table lists every item with search and filters.

## Finding items

- **Search** — free-text box to find items by title.
- **Kind filter** — Book, Video, Episode, Live, Podcast, Article.
- **Status filter** — Draft, Scheduled, Published, Archived.

The table shows each item's thumbnail and title, kind, status (color-coded), 30-day read count, and
last-updated time, with an **Edit** link per row.

## Bulk actions

Select rows with the checkboxes to **Publish**, **Archive**, or **Delete** several items at once
(Delete requires the matching permission). You can also **Export CSV** for an offline copy, and
manage your **Content Types** (key, label, enabled) in a dialog.

## Creating an item

Click **New** to open the create form:

| Field | Notes |
|---|---|
| Title | Required |
| Slug | Optional; auto-derived from the title (lowercase, dashes) |
| Kind | book, video, episode, live, podcast, article |
| Initial status | draft, scheduled, published |
| Metadata | Optional structured data (e.g. language) |

After saving you can upload a cover image from the edit page.

## Editing an item

The editor uses tabs so each section saves independently: **Básicos** (title, slug, kind, status),
**Media** (cover/poster uploads), **Cast & crew**, **SEO**, **Disponibilidad** (availability), and
**Publicación** (scheduling). A sidebar shows the cover, current status, and engagement metrics. If
another operator edits the same item at once, you'll see a concurrent-edit warning so no work is lost.
