# Flow: Google AdSense

## Overview

This flow documents how to integrate Google AdSense with your websites through the 1Platform API. It covers OAuth authorization, connecting AdSense accounts, viewing earnings (overview, by page, custom reports), managing sites, monitoring alerts and policy issues, and disconnecting or repairing connections.

**Use case:** Connect your users' websites to Google AdSense, monitor ad earnings and performance, track policy issues and alerts, and manage the full AdSense lifecycle — all through a unified API.

**End result:** A fully connected Google AdSense account with access to earnings data, site management, alerts monitoring, and policy issue tracking.

---

## Prerequisites

| Requirement | Details |
|---|---|
| App API Key | Application key (`ak-...`) provided when registering your app |
| User API Key | User key (`sk-...`) obtained when creating the user |
| Website registered | A website must be registered via the Manage Websites flow |
| Google AdSense account | An active Google AdSense account is required |

### Required headers for all requests

```
Authorization: Bearer $APP_TOKEN
x-user-token: $USER_TOKEN
Content-Type: application/json
```

## Step-by-step flow

### Step 0 — Get App Token

**Goal:** Authenticate the application and obtain the app JWT token.

**Endpoint:** `POST /api/v1/auth/token`

**Auth:** None (this is the entry point)

```bash
curl -X POST "$BASE_URL/auth/token" \
  -H "Content-Type: application/json" \
  -d '{"apiKey": "'$APP_API_KEY'"}'
```

**Response (200):**

```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 1800
  },
  "msg": "App token generated successfully"
}
```

:::note
Save `data.access_token` as `$APP_TOKEN`. Expires in 30 minutes.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Invalid API key or inactive app | Verify that `apiKey` is correct and the app is active |
| 422 | Missing or malformed `apiKey` field | Send `{"apiKey": "..."}` in the body |
| 429 | Rate limit exceeded (10 req/min) | Wait and retry |

```json
{
  "success": true,

    "expires_in": 1800
```

:::note
Save `data.access_token` as `$USER_TOKEN`. Expires in 30 minutes.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|

### Step 2 — Start OAuth Authorization

**Goal:** Initiate the Google OAuth flow for AdSense. The API returns an authorization URL that the user must visit in their browser to grant access.

**Endpoint:** `GET /api/v1/adsense/oauth/authorize`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X GET "$BASE_URL/adsense/oauth/authorize?website_id=$WEBSITE_ID" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN"
```

**Query parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `website_id` | string | Yes | Website ID to authorize AdSense for |

**Response (200):**

```json
{
  "success": true,
  "data": {
    "authorization_url": "https://accounts.google.com/o/oauth2/v2/auth?client_id=...&redirect_uri=...&scope=https://www.googleapis.com/auth/adsense.readonly&state=507f1f77bcf86cd799439011",
    "state": "507f1f77bcf86cd799439011"
  },
  "msg": "Authorization URL generated successfully"
}
```

:::warning
The user must open the `authorization_url` in their browser, sign in with their Google account, and grant AdSense permissions. The OAuth callback is handled automatically by 1Platform. After authorization, use Step 3 to verify the OAuth status before connecting.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 401 | Missing or invalid authentication token | Regenerate tokens (Steps 0 and 1) |
| 422 | Validation Error — missing or invalid `website_id` | Provide a valid `website_id` query parameter |
| 429 | Rate limit exceeded | Wait and retry |

### Step 4 — Connect AdSense

**Goal:** Activate the AdSense connection for the website after OAuth has been authorized.

**Endpoint:** `POST /api/v1/adsense/connect`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/adsense/connect?website_id=$WEBSITE_ID" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN"
```

**Query parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `website_id` | string | Yes | Website ID to connect AdSense for |

**Response (200):**

```json
{
  "success": true,
  "data": {
    "account_id": "pub-1234567890123456",
    "account_name": "Ejemplo AdSense Demo",
    "publisher_id": "pub-1234567890123456",
    "status": "active",
    "timezone": "America/Guatemala",
    "site_state": "READY",
    "site_auto_ads_enabled": true,
    "connected_at": "2026-04-06T14:35:00Z",
    "error_message": null,
    "last_sync_at": "2026-04-06T14:35:00Z",
    "oauth_connected": true,
    "oauth_email": "publisher@example.com"
  },
  "msg": "AdSense connected successfully"
}
```

:::note
The response is an `AdSenseStatusResponse` object. Key fields:
- `status` — Connection status (`"active"`, `"disconnected"`, `"error"`)
- `publisher_id` — Your AdSense publisher ID (format: `pub-NNNN`)
- `site_state` — Google's approval state for this site (e.g., `"READY"`, `"REQUIRES_REVIEW"`)
- `site_auto_ads_enabled` — Whether auto ads are enabled for this site
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 400 | OAuth not authorized | Complete the OAuth flow first (Steps 2 and 3) |
| 409 | Already active or invalid state transition | Check current status with `GET /adsense/status`. The connection may already be active |
| 422 | Validation Error — missing or invalid `website_id` | Provide a valid `website_id` query parameter |

### Step 11 — Custom Earnings Report

**Goal:** Generate a custom earnings report with specific date ranges and dimensions.

**Endpoint:** `POST /api/v1/adsense/earnings/report`

**Auth:** `Authorization: Bearer $APP_TOKEN` + `x-user-token: $USER_TOKEN`

```bash
curl -X POST "$BASE_URL/adsense/earnings/report?website_id=$WEBSITE_ID" \
  -H "Authorization: Bearer $APP_TOKEN" \
  -H "x-user-token: $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "start_date": "2026-03-01",
    "end_date": "2026-03-31",
    "dimensions": ["DATE", "COUNTRY"]
  }'
```

**Query parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `website_id` | string | Yes | Website ID with an active AdSense connection |

**Request body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `start_date` | string | Yes | Start date in `YYYY-MM-DD` format |
| `end_date` | string | Yes | End date in `YYYY-MM-DD` format |
| `dimensions` | array | No | Array of dimension strings (e.g., `"DATE"`, `"COUNTRY"`, `"AD_UNIT"`). Default: `["DATE"]` |
| `metrics` | array | No | Array of metric strings (e.g., `"ESTIMATED_EARNINGS"`, `"CLICKS"`, `"IMPRESSIONS"`). Default: `["ESTIMATED_EARNINGS", "CLICKS", "IMPRESSIONS"]` |

**Response (200):**

```json
{
  "success": true,
  "data": {
    "headers": [
      { "name": "DATE", "type": "DIMENSION" },
      { "name": "COUNTRY", "type": "DIMENSION" },
      { "name": "ESTIMATED_EARNINGS", "type": "METRIC_CURRENCY", "currency_code": "USD" },
      { "name": "PAGE_VIEWS", "type": "METRIC_TALLY" },
      { "name": "CLICKS", "type": "METRIC_TALLY" }
    ],
    "rows": [
      ["2026-03-01", "US", "12.50", "3200", "85"],
      ["2026-03-01", "MX", "3.25", "1100", "28"],
      ["2026-03-02", "US", "14.00", "3500", "92"],
      ["2026-03-02", "GT", "2.10", "800", "15"]
    ]
  },
  "msg": "Custom report generated successfully"
}
```

:::note
The `headers` array describes each column in the `rows`. Each row is an array of string values corresponding to the headers in order. Currency values are returned as string representations of decimal numbers.
:::

**Common errors:**

| Code | Cause | Solution |
|---|---|---|
| 422 | Validation Error — missing fields or invalid date range | Ensure `start_date`, `end_date`, and `dimensions` are present and valid |

### Step 14 — Disconnect AdSense

## Idempotency and best practices

### Idempotency

:::note
All `GET` endpoints in this flow are naturally idempotent — they can be called repeatedly without side effects. The `POST /adsense/connect` endpoint is guarded by state transitions: calling it when already active returns a `409` conflict, preventing duplicate connections.
:::

### Retries

- If you receive **429** (rate limit), wait the indicated time and retry.
- If you receive **500** or a **timeout**, check the current status with `GET /adsense/status` before retrying a connection or disconnect operation.
- For **401** (expired token), regenerate the token and retry the original request.

### Timeouts

:::warning
OAuth authorization and connection operations involve external calls to Google APIs. Configure your HTTP client with at least **30 seconds** timeout for `POST /adsense/connect` and `POST /adsense/sites/sync`. Earnings report endpoints may take longer for large date ranges — use **60 seconds** timeout for `POST /adsense/earnings/report`.
:::

### Tokens

- Both tokens (app and user) expire in **30 minutes** (1800 seconds). Implement automatic renewal before expiration.
- Both tokens are required on all AdSense endpoints.

### Important validations

- The website must be registered via the Manage Websites flow before connecting AdSense.
- OAuth must be completed (Steps 2-3) before connecting (Step 4).
- The `website_id` query parameter is required on every AdSense endpoint.
- Disconnect (Step 14) before revoking OAuth (Step 15) if the connection is active.
- The repair endpoint (Step 16) is admin-only and should not be called from regular user flows.
