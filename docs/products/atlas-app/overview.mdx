# Atlas App

The **Atlas App** is the white-label native application your audience installs on iOS, Android and
TV. Each tenant gets its **own branded build** — app name, icon, splash, colors and bundle identity —
delivering an in-app reader (EPUB, PDF, MOBI), media playback, offline downloads, watchlist,
achievements, and membership-aware access.

It follows a **reader-app model**: membership is purchased and managed on the web, and the app honors
your entitlements without native in-app purchases — keeping a single source of truth and store
compliance.

> This section is for **operators** offering a branded app to their audience — no code required.
> The app consumes the [Atlas API](/docs/saas/atlas-api/overview).

## What your audience gets

| Area | Capability |
|---|---|
| **In-app reader** | EPUB / PDF / MOBI with fonts, themes, bookmarks, and progress sync. |
| **Browse & search** | Catalog discovery, filters, authors, genres, full-text search. |
| **Playback** | Video (adaptive streaming) and audio with resume and speed control. |
| **Offline** | Download for offline reading; manage storage and quota. |
| **Account** | Magic-link sign-in, profile, security (2FA / biometric), devices. |
| **Membership** | View plan and benefits; manage on the web. |
| **Library** | Watchlist, collections, continue reading, achievements. |
| **Personalization** | Theme, reader defaults, language, push preferences. |
| **Parental controls** | Age gates, restrictions, child accounts. |

## White-label by design

- **Build-time** (`brand.json`) — app name, URL scheme, bundle/package id, icon, splash, fallback
  theme, and feature flags (downloads, biometric, push, audio, video, manga).
- **Runtime** — branding, feature flags, supported languages, and store links resolved per tenant.
- **Isolation** — all member data is scoped per tenant; an app binary only ever sees its own tenant.
