# Testing

Practical recipes for integrating without touching real money or real users.

## Sandbox tenant

QA base URL: `https://api-qa.1platform.pro/api/v1`. Request credentials via
your account manager — you'll receive an `ak-` (app) and `sk-` (user) key
scoped to QA only. See [Environments](./environments) for the differences.

## Test card numbers

The integrated payment provider's sandbox accepts the standard Visa test
numbers:

| Number | Outcome |
|---|---|
| `4111 1111 1111 1111` | Approved (`status: "approved"`) |
| `4000 0000 0000 0002` | Declined (`status: "denied"`) |
| `4000 0000 0000 0119` | Expired (`status: "expired"`) |

Use any future expiry date and any 3-digit CVV.

## Replaying webhooks against localhost

Three steps:

1. **Tunnel localhost.** Run `ngrok http 3000` (or `smee`, `cloudflared`, etc.)
   to expose your local handler at a public URL.
2. **Register the public URL.**
   ```bash
   curl -X POST "$BASE_URL/webhooks/config/domains" \
     -H "Authorization: Bearer $APP_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"domain": "abc123.ngrok-free.app"}'

   curl -X PUT "$BASE_URL/webhooks/config/events" \
     -H "Authorization: Bearer $APP_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{
       "on_approved": { "url": "https://abc123.ngrok-free.app/webhooks/payments", "method": "POST" }
     }'
   ```
3. **Trigger a transaction** in QA with `4111 1111 1111 1111` and observe the
   webhook hit your local handler.

## Replaying captured payloads offline

Once you've captured one real payload (headers + body), you can replay it
against your handler without going through the API:

```bash
curl -X POST http://localhost:3000/webhooks/payments \
  -H "X-Webhook-Signature: sha256=..." \
  -H "X-Webhook-Timestamp: 1714389600" \
  -H "Content-Type: application/json" \
  --data-binary @captured.json
```

:::caution Replays expire after 5 minutes
Captured payloads pass signature verification only inside the replay window.
If you need to replay the same payload over and over (e.g. while debugging
your handler), temporarily widen the window in your **test** verifier — never
in prod.
:::

## Smoke tests

Minimum smoke test for a new integration:

1. Mint app + user tokens (`POST /auth/token`, `POST /users/token`).
2. `GET /users/billing` and confirm `data.billing.payment_gateway_id` is
   non-null. If it's null, see [troubleshooting](./troubleshooting).
3. `POST /users/transactions` with `merchant_reference` set to a unique
   string.
4. Open the returned `payment_url` in a browser; pay with the approved test
   card.
5. Verify your webhook handler receives `event: on_approved` with the same
   `merchant_reference`.
6. Confirm signature verification passes and your handler dedupes correctly
   on a forced replay.

If any step fails, the corresponding [troubleshooting](./troubleshooting)
entry should explain what to check.
