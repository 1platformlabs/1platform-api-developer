# Primeros pasos

Esta guía te lleva desde cero hasta tu primera respuesta autenticada contra la **1Platform API**.

## 1. Obtén tus tokens

La 1Platform API usa un modelo de **dos tokens**:

- **Token de aplicación** (`ak-...`) — identifica tu aplicación. Se envía como `Authorization: Bearer ak-...`.
- **Token de usuario** (`sk-...`) — identifica al usuario final que actúa. Se envía como `x-user-token: sk-...`.

Gestiona las claves de aplicación desde el [Dashboard → claves de API](/docs/products/dashboard/api-keys). Los tokens de
usuario se generan a través del flujo de autenticación (consulta
[Onboarding de usuarios](/docs/saas/1platform-api/flows/user-onboarding)).

## 2. Elige un entorno

| Entorno | URL base |
|---|---|
| Producción | `https://api.1platform.pro/api/v1` |
| QA / sandbox | `https://api-qa.1platform.pro/api/v1` |

Detalles completos en [Entornos](/docs/saas/1platform-api/reference/environments).

## 3. Realiza tu primera llamada

```bash
curl https://api-qa.1platform.pro/api/v1/users/me \
  -H "Authorization: Bearer ak-your-app-token" \
  -H "x-user-token: sk-the-user-token"
```

Una respuesta exitosa usa el sobre estándar:

```json
{ "success": true, "data": { /* ... */ }, "msg": "OK" }
```

Si una llamada requiere créditos que no tienes, recibirás `402 Payment Required`; si superas un
límite de uso, `429 Too Many Requests`. Consulta [Códigos de error](/docs/saas/1platform-api/reference/error-codes)
y [Límites de uso](/docs/saas/1platform-api/reference/rate-limits).
