# Módulos

El dashboard es **modular**. Cada capacidad es un módulo que puede habilitarse o deshabilitarse por workspace, de modo que dos workspaces que ejecutan el mismo producto pueden ofrecer conjuntos de funciones muy diferentes.

## Cómo aparecen los módulos

Los módulos habilitados se muestran como entradas en la barra lateral izquierda. Para cada módulo, la configuración del workspace controla:

- **Si está habilitado** — activado o desactivado para el workspace.
- **Su etiqueta e ícono** — cómo se lee en la barra lateral.
- **Su orden** — dónde se ubica en la lista.
- **Quién puede verlo** — requisitos de permisos, y si es solo para administradores.

Algunos módulos también tienen **sub-elementos** que aparecen anidados debajo de ellos.

## Módulos habilitados comúnmente

| Módulo | Qué hace |
|---|---|
| **Dashboard** | Estadísticas del workspace, tendencia de uso y actividad. |
| **Cobros** | Saldo, recargas y transacciones. |
| **Facturación** | Emitir y dar seguimiento a facturas; gestionar empresas emisoras. |
| **Dominios** | Conectar y monitorear dominios personalizados. |
| **Onboarding** | La lista de verificación de configuración guiada. |
| **Configuración** | Perfil, workspace, seguridad y más. |
| **Notificaciones** | Alertas del workspace. |
| **Registros** (administración) | Historial de solicitudes API. |

## Módulos opcionales

Se pueden activar módulos adicionales cuando un workspace los necesita — por ejemplo sitios web, herramientas, plugins, referidos, tareas, tickets y preguntas frecuentes. Permanecen ocultos hasta que se habilitan, manteniendo la barra lateral de cada workspace enfocada en lo que realmente usa.

## Feature flags

Junto con los módulos, los **feature flags** permiten desplegar capacidades de forma gradual a workspaces específicos. Así es como una funcionalidad nueva puede ser probada por algunos workspaces antes de activarse en todas partes.

## Configurar módulos

Los módulos y los feature flags forman parte de la configuración por workspace. Para habilitar o reordenar un módulo, contacta a tu administrador del workspace o a soporte. La apariencia y la marca se configuran de la misma manera — consulta [Marca y apariencia](/docs/products/dashboard/branding-appearance).
