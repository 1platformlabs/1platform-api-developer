# Rate limits

Rate limits are enforced via [slowapi](https://slowapi.readthedocs.io/) on
every protected endpoint. Going over the limit returns `429 Too Many Requests`.

## Rate-limit key

The rate-limit key is the SHA-256 of `(Authorization header + x-user-token header)`.
When both headers are absent, the API falls back to the client IP address.

## Per-endpoint limits

| Constant | Value | Endpoints |
|---|---|---|
| `HEALTH_LIMIT` | 100 / min | `/health` |
| `AUTH_LIMIT` | 10 / min | `POST /auth/token`, `POST /users/token` |
| `READ_LIMIT` | 50 / min | All `GET` endpoints |
| `WRITE_LIMIT` | 20 / min | All `POST`/`PUT`/`PATCH`/`DELETE` (except specials below) |
| `KEYWORD_LIMIT` | 10 / min | `/posts/keywords/*` |
| `CONTENT_LIMIT` | 5 / min | `/posts/content/*` |
| `INDEXING_LIMIT` | 20 / min | `/posts/indexing/*` |

Limits are identical across QA and production.

## Handling 429

```python
import time
import httpx

def call_with_backoff(client: httpx.Client, *args, **kwargs) -> httpx.Response:
    for attempt in range(3):
        r = client.request(*args, **kwargs)
        if r.status_code != 429:
            return r
        retry_after = int(r.headers.get("Retry-After", "60"))
        time.sleep(retry_after)
    return r  # exhausted retries
```

:::tip Cache tokens to stay under AUTH_LIMIT
The 10 / min limit on `auth/token` and `users/token` is the most common reason
to hit a 429 during development. Cache tokens for `expires_in - 60s` and only
mint a fresh one when the cached value is about to expire. See the
[token caching tip](../flows/payments-and-subscriptions#token-caching-pattern).
:::
