# Settings — Security

The **Security** tab lives under Settings, between Workspace and Billing.

## Coming soon

This pane is a **planned feature**. When it ships it will gather account-security controls in one place, including:

- **Sign-in management** — review and manage how you access the workspace.
- **Additional verification** — extra protection on top of sign-in.
- **Active sessions** — see where you're currently signed in.
- **Login history** — a record of recent sign-ins.

Until then, the tab shows a clear "coming soon" message.

## How sign-in works today

Your workspace uses **magic-link sign-in only** — there are no passwords to set, rotate, or recover. To sign in, you request a one-time link by email and click it; the link is single-use and time-limited. Because there's no password, there's nothing to be stolen or reused.

See [Getting started](/docs/products/dashboard/getting-started) for the full sign-in flow.

## Workspace-level security

Administrators can rotate the workspace API key, which immediately invalidates existing integration tokens. That control lives on the admin App settings screen — see [API keys](/docs/products/dashboard/api-keys) and [Admin & impersonation](/docs/products/dashboard/admin-impersonation).
