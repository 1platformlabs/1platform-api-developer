# Atlas API

La **Atlas API** es una plataforma de entrega de contenido independiente y multitenant de 1Platform Labs. Úsala para
construir tu propio ecosistema con tu marca — una biblioteca, un servicio de streaming, una red de podcasts o una plataforma
de publicación — con gestión de catálogo, autenticación, entitlements y entrega de medios, todo aislado
por tenant.

> Atlas es **independiente** de la 1Platform API: modelo de autenticación distinto, sus propios datos, sin internals
> compartidos. Para la API de la plataforma central, consulta [1Platform API](/docs/saas/1platform-api/overview).

## Cómo difiere de la 1Platform API

| | 1Platform API | Atlas API |
|---|---|---|
| **Autenticación** | Dos tokens (`ak-` + `sk-`) | **JWT bearer** (tras el magic link) + claves de integración |
| **Tenant** | Por `app` / `user` | Resuelto por `Host` / `X-Tenant-Slug` |
| **Forma de error** | `{ success, data, msg }` | `{ code, message }` |
| **URL base** | `api.1platform.pro/api/v1` | `atlas-api.1platform.pro/api/v1` |

## Flujos de integración principales

- [Autenticación](/docs/saas/atlas-api/authentication) — magic link → JWT, refresco, sesiones.
- [Claves de integración](/docs/saas/atlas-api/integration-keys) — acceso de servicio a servicio.
- [Resolución de tenant](/docs/saas/atlas-api/tenant-resolution) — resuelve un tenant por host o slug.
- [Exploración del catálogo](/docs/saas/atlas-api/catalog-browsing) — lista, obtén por slug, busca.
- [Entrega de contenido](/docs/saas/atlas-api/content-delivery) — tickets de reproducción, URLs firmadas, streaming.
- [Entitlements](/docs/saas/atlas-api/entitlements) — concede, revoca y verifica acceso (scopes: item, series, plan, live).
- [Webhooks](/docs/saas/atlas-api/webhooks) — recibe eventos de entitlement (firmados, idempotentes).
