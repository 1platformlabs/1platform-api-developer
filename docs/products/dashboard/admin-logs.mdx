# Registros de actividad y solicitudes

La pantalla **Registros de solicitudes API** (administración) muestra cada solicitud realizada a tu workspace. Es un lugar útil para confirmar que las integraciones funcionan y para detectar cualquier fallo.

## La tabla de registros

Cada fila registra una sola solicitud:

| Columna | Qué muestra |
|---|---|
| **Hora** | Cuándo llegó la solicitud. |
| **Método** | El tipo de solicitud (leer, crear, actualizar, eliminar). |
| **Endpoint** | Qué parte de la API se llamó. |
| **Estado** | El código de resultado — verde para éxito, ámbar para problemas del cliente, rojo para errores del servidor. |
| **Duración** | Cuánto tardó la solicitud, en milisegundos. |

Los códigos de estado están codificados por color para que puedas detectar problemas de un vistazo.

## Filtrado

Acota la lista con los controles encima de la tabla:

- **Buscar** — por endpoint o método.
- **Método** — todos, o un solo tipo de solicitud.
- **Rango de tiempo** — hoy, últimos 7 días o últimos 30 días.

## Exportar

Usa **Exportar** para descargar el registro y conservarlo o analizarlo en mayor profundidad.

## Estados vacío y de carga

Si el workspace aún no ha recibido tráfico, un estado vacío explica que las entradas aparecerán en cuanto lleguen solicitudes. Mientras el registro se carga, se muestran marcadores de posición brevemente.

## Dónde se conecta

El feed **Actividad reciente** del inicio del dashboard es una vista más ligera de los mismos eventos; elegir **Ver todo** allí te lleva a este registro completo. Consulta [Inicio del dashboard](/docs/products/dashboard/dashboard-home).
