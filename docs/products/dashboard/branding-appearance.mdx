# Branding & appearance

The dashboard is **white-label**: every workspace runs the same product but looks like *your* brand. Branding, colors, typography, and layout are configured per workspace and applied automatically when the dashboard loads.

## Brand identity

Each workspace defines its own brand:

| Setting | What it controls |
|---|---|
| **Name & short name** | The product name shown in the interface and browser tab. |
| **Logo** | Your logo, with separate light and dark variants. |
| **Favicon** | The little icon in the browser tab. |
| **Social image** | The preview image when a link is shared. |
| **Support email** | Where members go for help. |
| **Legal links** | Terms, privacy, and cookie pages. |

## Appearance

The look is themeable per workspace:

- **Color palette** — primary, secondary, and accent colors, plus success, warning, and error tones.
- **Theme mode** — light, dark, or automatic (follows the visitor's system setting).
- **Typography** — the fonts used throughout.
- **Surfaces** — background and card colors, dividers, and component styling.

Because these are applied at load time, changing them re-skins the whole dashboard without any rebuild.

## Layout options

A few interface toggles are set per workspace:

- Whether the sidebar starts collapsed.
- Whether the **language switcher** appears.
- Whether the **theme toggle** appears.
- Whether the **notifications bell** appears.

## Who configures it

Branding and appearance are part of each workspace's configuration, applied centrally. To request a change to your brand, colors, or layout, contact your workspace administrator or support. Which features show in the sidebar is a related setting — see [Modules](/docs/products/dashboard/modules).
