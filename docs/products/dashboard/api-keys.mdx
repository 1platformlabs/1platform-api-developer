# Claves API

Las claves API permiten que otros sistemas se conecten a tu workspace en tu nombre. Las encontrarás en la pestaña **API** de Configuración, y los administradores gestionan la clave del workspace en la pantalla de administración Configuración de la app.

## La pestaña API

El panel **Acceso API** muestra tu clave actual (parcialmente enmascarada por seguridad) junto con detalles como cuándo se creó y cuándo se usó por última vez. Hay dos acciones disponibles:

- **Copiar** — copia la clave a tu portapapeles para que puedas pegarla en una integración. Aparece una confirmación; si la copia falla, se te pide seleccionar la clave manualmente.
- **Rotar** — reemplaza la clave actual por una nueva. Primero se te pide confirmar, porque rotar revoca la clave actual de inmediato.

## Rotar de forma segura

La rotación es **inmediata e irreversible**. En el momento en que rotas:

1. La clave antigua deja de funcionar al instante.
2. Se emite una nueva clave.
3. Cada integración que siga usando la clave antigua fallará hasta que la actualices.

Por eso, antes de rotar, asegúrate de poder actualizar la clave allí donde esté configurada justo después.

## Clave del workspace (administradores)

Los administradores del workspace gestionan la clave principal en la pantalla de administración Configuración de la app. Allí, **Rotar clave API** pide confirmación y luego muestra la nueva clave **una sola vez** — cópiala de inmediato, porque no se mostrará de nuevo. La rotación invalida cada token de integración activo, así que actualiza tus integraciones enseguida. Consulta [Administración e impersonación](/docs/products/dashboard/admin-impersonation).

## Buenas prácticas

- Trata las claves como contraseñas — nunca las compartas ni las publiques.
- Rota de inmediato si una clave pudo haber quedado expuesta.
- Actualiza cada integración inmediatamente después de rotar.
