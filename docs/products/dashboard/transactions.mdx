# Transactions

Your transaction history is the running record of every credit movement in the workspace. You'll find it in the **Recent transactions** section of the [Billing & credits](/docs/products/dashboard/billing-credits) page.

## What each row means

Every transaction shows:

- **Description** — a short summary of what happened.
- **Date** — when it occurred.
- **Type** — the kind of movement (see the table below).
- **Amount** — the value involved.
- **Status** — for example *paid* or *refunded*.

## Transaction types

| Type | What it represents |
|---|---|
| **Top-up** | Credits you added to your balance. |
| **Subscription** | A recurring charge against the workspace. |
| **Usage / invoice** | Credits consumed by the work your workspace did. |
| **Refund** | Credits returned to your balance. |

## Reading the trend

The dashboard home complements this list with a 12-week chart, a monthly average, and a peak-week figure, so you can spot whether spend is rising or falling. See [Dashboard home](/docs/products/dashboard/dashboard-home).

## Empty and loading states

When the list is still loading you'll see placeholder rows. If there's no history yet, an empty state explains that charges, top-ups, and usage entries will appear here as soon as you start billing. This keeps the page understandable even on a brand-new workspace.
