# Domains

The **Domains** page lets you connect, manage, and monitor the domains used across your workspace — the addresses where your content is published.

## The domains table

Each connected domain appears as a row showing:

- **Domain** — the domain name.
- **Note / registrar** — where the domain is managed.
- **Status** — its current state (see below).
- **Renews** — the next renewal date, when known.

## Domain statuses

| Status | What it means |
|---|---|
| **Active** | The domain is connected and working. |
| **Pending DNS** | The domain is being set up or transferred and isn't live yet. |
| **Expired** | The domain has lapsed and needs attention. |

## Adding a domain

Select **Add domain** to connect or register a new one. When you have no domains yet, an empty state invites you to connect or register your first domain so you can start publishing.

## If the list won't load

If domains can't be fetched, the page shows an error with a **Retry** button — select it to try loading again. Loading placeholders appear briefly while data is fetched.

## Where this fits

Domains are part of your workspace setup. The onboarding checklist nudges you to connect your first domain early — see [Onboarding](/docs/products/dashboard/onboarding) — and the dashboard home counts your active domains as a headline stat.
