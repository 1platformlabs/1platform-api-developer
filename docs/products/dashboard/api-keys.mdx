# API keys

API keys let other systems connect to your workspace on your behalf. You'll find them under the **API** tab in Settings, and administrators manage the workspace key on the admin App settings screen.

## The API tab

The **API access** pane shows your current key (partially masked for safety) along with details such as when it was created and when it was last used. Two actions are available:

- **Copy** — copies the key to your clipboard so you can paste it into an integration. A confirmation appears; if copying fails, you're prompted to select the key manually.
- **Rotate** — replaces the current key with a new one. You're asked to confirm first, because rotating revokes the current key immediately.

## Rotating safely

Rotation is **immediate and irreversible**. The moment you rotate:

1. The old key stops working at once.
2. A new key is issued.
3. Every integration still using the old key will fail until you update it.

So before rotating, make sure you can update wherever the key is configured right after.

## Workspace key (admins)

Workspace administrators manage the primary key on the admin App settings screen. There, **Rotate API key** asks for confirmation, then shows the new key **once** — copy it immediately, because it won't be shown again. Rotating invalidates every active integration token, so update your integrations straight away. See [Admin & impersonation](/docs/products/dashboard/admin-impersonation).

## Good practice

- Treat keys like passwords — never share or post them publicly.
- Rotate promptly if a key may have been exposed.
- Update every integration immediately after rotating.
