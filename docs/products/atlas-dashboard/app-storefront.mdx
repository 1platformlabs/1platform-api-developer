# App Storefront

The **Mobile App** screen (`/admin/app`) configures your companion app for phones and tablets. It is
organized into sections; turn on what you need and leave the rest off.

## App features

Enable the app and pick which capabilities it offers: **Downloads**, **Biometric** unlock, **Push**
notifications, **Audio**, **Video**, and **Manga**. You can also restrict certain capabilities
(downloads, audio, video, manga) to be **app-exclusive**, so they're only available inside the app.

## Store links

Set the public listing URLs for the **App Store** and **Google Play**, your minimum supported version,
and a custom URL scheme used for deep links into the app.

## Releases

The **Releases** section holds the metadata for your app listing: tagline, description, category,
content rating, support URL, "what's new" notes, and up to five screenshots (each with image and alt
text, reorderable). Published builds (version, size, update date for each platform) appear read-only —
they come from your release pipeline, not this form.

## Updates

The **Update** section controls how the app prompts users to upgrade: the latest and minimum supported
versions, download links, and a **force update** option for users on outdated versions.

## In-app reader

The **Reader** section enables reading inside the app and selects which formats it supports (for
example **EPUB**, **PDF**, **MOBI**).

## Measurement

The **Measurement** section enables app analytics and selects which events to capture (such as app
open, content open, content finish, download, and paywall views). Results appear in
[App Analytics](/docs/products/atlas-dashboard/app-analytics). You can require user consent before any
measurement is collected.
