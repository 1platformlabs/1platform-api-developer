# Watchlist & library

Every reader gets a personal library in the Atlas App that brings together what they're reading, what they've saved, and what they've downloaded.

## What your audience gets

The **My library** tab has five buckets, each loaded only when opened so the tab stays fast:

- **Continue** — titles in progress, with a resume progress bar, drawn from the reader's saved positions.
- **Downloads** — titles downloaded to the device; tapping one opens the full download manager.
- **Favorites** — titles the reader marked as favorites.
- **Lists** — custom saved lists (for example "For later").
- **Finished** — titles the reader has completed.

### Saving titles

From a title's page or a card, the reader uses a **save** control: a quick menu of buckets (Favorites, For later, custom lists) with a checkmark showing what's already saved. Saves are instant and reflected across the app — the same saved entries power both the checkmarks and the library buckets. Saving and removing are optimistic, so the interface responds immediately.

## What you control

The library is a per-reader feature that works automatically; there's no per-tenant setup. It draws on:

- **Reading progress** — which feeds Continue and Finished (and depends on the in-app reader being on; see [Reader experience](/docs/products/atlas-app/reader-experience)).
- **Downloads** — which depend on the downloads feature flag; see [Offline downloads](/docs/products/atlas-app/offline-downloads).

All library and watchlist data is scoped to each reader within your tenant, so it's private to them.
